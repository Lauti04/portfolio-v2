# CLAUDE.md

## Project

This is the personal portfolio of Lautaro Johnston, a Junior Full Stack Developer based in Benalmádena, Málaga, Spain.

The portfolio must feel professional, polished, modern, responsive and carefully crafted. It should communicate that Lautaro is a junior developer with real project experience, strong learning ability and good technical judgment.

## Main goal

Build a high-quality portfolio that helps Lautaro get developer job opportunities in Spain and remote/international roles.

The site should stand out through:

* Strong visual design
* Excellent responsive behavior
* Clean and reusable React architecture
* Good accessibility
* Smooth but tasteful interactions
* Multilingual support
* Beautiful light and dark themes
* Clear technical storytelling
* Fast performance

## Stack

Use:

* React
* TypeScript
* Vite
* Tailwind CSS
* ESLint
* Prettier

Do not add dependencies unless they clearly improve the project. Explain why before adding any dependency.

## Development rules

* Do not implement large features without first proposing a plan.
* Do not create generic template-looking sections.
* Do not use placeholder text when real information is available.
* Ask before inventing missing content.
* Keep components small, readable and reusable.
* Prefer simple solutions over over-engineered code.
* Avoid repeated markup and duplicated logic.
* Use semantic HTML.
* Use accessible interactive elements.
* Use TypeScript types for structured content.
* Keep portfolio content in typed data files when appropriate.
* Keep design tokens centralized when possible.
* Use clear file and component names.
* All code comments must be in English.
* Do not write unnecessary comments.
* Run lint/build before considering work complete.
* Do not modify unrelated files.
* Do not change the project direction without asking.

## Design direction

The design should be minimal, premium and modern, not childish or overloaded.

Preferred style:

* Dark/light mode with high-quality palettes
* Subtle gradients or background textures
* Clean typography
* Good spacing and hierarchy
* Smooth microinteractions
* Elegant hover states
* Responsive layouts for mobile, tablet and desktop
* Optional custom cursor only if it does not hurt usability or accessibility
* No excessive animations
* No heavy effects that reduce performance

The portfolio should feel more like a polished developer product than a basic student portfolio.

## Sections

Required sections:

* Hero

* Experience

Nice-to-have sections:

* Featured project deep dive

## Internationalization

Support multiple languages using a clean i18n structure.

Minimum languages:

* Spanish
* English

Future-friendly languages:

* French
* German
* Portuguese
* Italian
* Dutch
* Catalan

Do not hardcode text directly inside components if it belongs to page content.

## Theme system

Implement light and dark themes properly.

Requirements:

* Theme switcher
* Persist selected theme
* Respect system preference when no user choice exists
* Avoid color duplication across components
* Ensure contrast and readability in both themes

## Quality checklist

Before marking a task as done:

* The page works locally
* TypeScript has no avoidable errors
* Build passes
* Layout works on mobile, tablet and desktop
* Links work
* Keyboard navigation is not broken
* Content is not duplicated unnecessarily
* Code is readable and organized
* No unrelated files were modified

## Git workflow

* Work in small steps.
* Make changes that are easy to review.
* Summarize what changed after each task.
* Suggest commits only after lint/build passes.
